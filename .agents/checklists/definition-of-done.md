# Definition of done

- Requirement or bug is implemented with minimal scope.
- Code follows existing patterns.
- Public API behavior is stable or change is explicitly documented.
- Request/response DTOs and validation are updated.
- Auth and object-level authorization are correct.
- Persistence changes are safe and reviewed.
- Tests cover the changed behavior.
- Build and tests pass, or failures are explained.
- OpenAPI/docs are updated when contracts changed.
- No secrets, sensitive logs, or broad CORS changes introduced.
